# 指示書：仮牛「ふうか」の追加と体調ループの実装

## これは何の作業か

最小ループ（ホーム→探索→リザルト→翌日→ホーム）は完成済み。
現状は牛がいないため魔力が固定でゴールドが積み上がるだけ。

今回の作業で以下を実現する：

1. 仮牛「ふうか」をセーブデータに追加し、魔力を体調から動的に計算する
2. 探索で集めた草のポイントが翌日の体調に影響し、体調が魔力を決める

これにより「探索→草を集める→翌日の魔力が変わる→より遠くへ行けるようになる（将来）」というゲームの基本ループが動く。

---

## ふうかのデータ（仮牛・固定値）

セーブデータ（yamayama_loop_v1）の `cows` 配列に以下を追加する。

```json
{
  "id": "cow_001",
  "name": "ふうか",
  "age": 48,
  "seed": 1234,
  "condition": 6,
  "quality": 2,
  "skill": "herdboys_eye"
}
```

- `age`：ゲーム内日数（1年=24日換算で2歳=48日）
- `seed`：ブチ模様の乱数シード（4桁整数）。既存の模様生成ロジックに渡す
- `condition`：体調内部値（1〜10。6=普通がスタート）
- `quality`：品質（1〜4。2=可がスタート）
- `skill`：スキル識別子。`"herdboys_eye"`（牧童の目 👦）

---

## 魔力の計算

common.js の `conditionToMagic(condition)` は実装済みのはず。
魔力は「全頭の体調から計算した魔力の合計」とする。
（現状1頭なので合算は将来の拡張に備えた設計）

```js
function calcTotalMagic(cows) {
  return cows.reduce((sum, cow) => sum + conditionToMagic(cow.condition), 0);
}
```

この関数を common.js に追加し、explore.html でのMAX_MANA計算に使う。

## 草ポイント→体調変動の実装

### 草ポイントの集計

探索終了時（リザルトモーダル表示タイミング）に、その日に獲得した草の合計ポイントを計算してセーブに持たせる。

| 草の種類 | ポイント |
|---|---|
| コモン（雑草） | 1 |
| アンコモン（良い牧草） | 3 |
| レア（薬草） | 体調ではなく品質ポイントに加算（今回は品質は固定なので無視でよい） |

合計ポイントを `yamayama_loop_v1` の `grassStock` に加算する（日をまたいで持ち越す）。

### 体調変動（翌日への計算）

date_change.html（または upkeep.html）で以下を処理する：

```
grassPointsPerCow = state.grassStock / state.cows.length  // 1頭あたりの草ポイント

3ポイント以下  → 体調を1下げる
4〜8ポイント  → 体調を維持
9ポイント以上 → 体調を1上げる
```

体調は 1〜10 の範囲でクランプする。
処理後 `grassStock` を 0 にリセットする。

**実装はif文でベタ書きでよい。JSON化はしない。**

### 体調変動後の表示

upkeep.html か date_change.html で、体調変動があった場合は一言表示する。
（例：「ふうかの体調が上がった！」「ふうかが少し元気がなさそうだ…」）
テキストは `t()` 経由で取得すること。

## スキル「牧童の目」の反映確認

`herdboys_eye`（👦 牧童の目）は explore.html 側で「ミスマッチ時の表示時間延長（1.5秒）」として実装済みのはず。
今回は「ふうかのスキルが herdboys_eye の場合に有効化」という形でスキル判定を繋げる。

```js
const hasHerdboyseye = state.cows.some(c => c.skill === "herdboys_eye");
// hasHerdboyseye が true の時だけ延長表示を有効にする
```
