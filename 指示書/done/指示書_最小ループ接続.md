# 指示書：最小ループの画面接続

## これは何の作業か

現在バラバラに存在する画面（ホーム／探索／管理）を、1本のループとして繋ぐ作業。
**新機能は作らない。** 既存のhtmlファイルの中身を活かしつつ、以下の順で行き来できるようにする。

```
ホーム → 探索（1エリア選択→神経衰弱）→ リザルト → 翌日 → ホーム（に戻る）
```

ストーリー・子牛・ショップは今回のスコープ外。親牛のみで1周できればゴール。

---

## 使う既存ファイル

- ホーム画面のベース：`home_v3_perspective.html`（発展後デザイン。まずはこれをベースにする）
- 探索パート：`hairal_memory_lv3.html`（スキル全部入り版）

上記を土台に、新しく `index.html`（または `game.html`）を1本作り、そこに画面遷移の骨格を実装する形でもよい。判断はお任せする。

---

## やってほしいこと

### 1. 画面遷移の骨格を作る
- ホーム画面から「探索へ行く」ボタンでエリア選択→探索パートへ遷移
- 探索パート終了（魔力を使い切る、または離脱ボタン）でリザルト画面へ
- リザルトから餌やりのＧＩＦ、「次の日へ」で魔力もらうのＧＩＦ、アップキープ処理（特になし）→ホーム画面に戻る
- 現時点ではエリアは複数出さなくてよい（ハイラル草原のみで可）。エリア選択UIは後回しでよい

### 2. データの受け渡し
- 体調・魔力・お金・品質など、画面をまたいで必要な値はセーブデータ（localStorage）経由で引き継ぐ
- **セーブデータは `{ version: 1, ... }` 形式で新規作成すること**（CLAUDE.md「itch.io公開を見据えた実装ルール」参照）
- 既存の `hairal_v3_lv3` 等のキーとは別に、ループ全体用の新しいキーを作ってよい（例：`yamayama_loop_v1`）。キー名は既存と重複しないこと

### 3. アップキープフェイズ（簡易でよい）
- 「翌日へ」を押した時に以下を判定：発情・難産・特になし、のいずれか
- イベントの内容は最小限（テキスト表示だけ）でよい。凝った演出は不要
- **イベント処理はif文でベタ書きでよい。JSON化はまだしない**（CLAUDE.md参照）

### 4. 画面表示の文言
- 新しく書く文字列は `t()` 経由で取得する形にしておく（辞書は `ja.json` のみでよい）
- 例：`t("go_explore")` のように。中身のja.jsonはこちらで用意するので、キー名だけ決めて実装してOK。使ったキー名は一覧で教えてほしい

### 5. スマホ前提
- 縦画面固定・タップ操作のみで動作確認すること
- ボタンは指でタップしやすいサイズを維持

## やらなくていいこと（スコープ外）

- ショップ
- 子牛の育成
- ストーリー・世界観要素
- 複数エリアの実装（ハイラル草原のみでOK）
- 難産の詳細処理（発生したことが分かればよい。処置選択は後回し）
- BGM・凝った演出

---

## 不明点があったら

実装を止めて、やましさんに確認すること。憶測で仕様を決めない。
（例：「難産の処理、こういう仕様で合ってますか？」等）

---

## 完成の目安

以下が1本のループとして繋がった状態で、スマホ実機（またはブラウザの縦画面表示）で1周動作確認できればこの作業は完了。

```
ホーム → 探索 → リザルト → 翌日（イベント判定） → ホーム（数値が引き継がれている）
```
