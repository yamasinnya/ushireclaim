# 指示書：品質システムの実装

## これは何の作業か

体調ループは実装済み。今回は「品質」の変動を実装する。
探索で薬草（レア）を集めると品質ポイントが貯まり、閾値を超えると品質が上がる。

---

## 品質のランク

| 内部値 | 表示名 |
|---|---|
| 4 | 優 |
| 3 | 良 |
| 2 | 可 |
| 1 | 劣 |

最低は1（劣）。0は存在しない。

---

## 品質ポイントの加算

探索終了時（草ポイント集計と同じタイミング）に、薬草（レア）の獲得枚数 × 5pt を品質ポイントに加算する。

セーブデータに `qualityPoint` を追加して持たせる（初期値0）。

```json
{ "version": 1, ..., "qualityPoint": 0 }
```

---

## 閾値と品質変動

`qualityPoint` が閾値に達したら品質を1段階上げ、`qualityPoint` を0にリセットする。
品質が4（優）の場合は上限なのでポイントをリセットするだけでよい。

**閾値は以下の定数で個別に定義すること（将来変更するため、同じ値でも1つにまとめない）：**

```js
const QUALITY_THRESHOLD_TO_RYO  = 10; // 可→良
const QUALITY_THRESHOLD_TO_YU   = 10; // 良→優
const QUALITY_THRESHOLD_TO_KA   = 10; // 劣→可
```

現時点では全て10（テスト用）。将来それぞれ異なる値に変更する予定があるため、定数を分けておくこと。

---

## 品質変動のタイミング

feeding.html（体調変動と同じ画面）で処理する。
品質が上がった場合は体調変動テキストと同様に一言表示する。
テキストは `t("quality_up")` を使うこと（キー追加が必要）。

品質は**上がるだけ**（下がる処理は今回実装しない）。
