# 指示書：数値調整とアップキープ名称変更

## これは何の作業か

テスト用の仮置き数値を本番値に変更する。
あわせて「アップキープ」という用語を「1日の始まり」に統一する。

---

## ①品質ポイントの閾値変更（common.js）

以下の定数を変更する：

```js
// 変更前
const QUALITY_THRESHOLD_TO_KA  = 10;  // 劣→可
const QUALITY_THRESHOLD_TO_RYO = 10;  // 可→良
const QUALITY_THRESHOLD_TO_YU  = 10;  // 良→優

// 変更後
const QUALITY_THRESHOLD_TO_KA  = 30;  // 劣→可
const QUALITY_THRESHOLD_TO_RYO = 50;  // 可→良
const QUALITY_THRESHOLD_TO_YU  = 80;  // 良→優
```

---

## ②💩による品質ポイントペナルティ変更（upkeep.html）

💩が4個の時の品質ポイント減少量を変更する：

```
変更前：poopCount >= 4 の時、毎日 qualityPoint -1
変更後：poopCount >= 4 の時、毎日 qualityPoint -5
```

qualityPointが0未満にならないようクランプすること（既存処理があれば確認）。

---

## ③「アップキープ」→「1日の始まり」への名称変更

ゲーム内で「アップキープ」と表示・使用されている箇所を全て「1日の始まり」に変更する。

対象は以下を含む（他にもあれば全て変更すること）：
- 画面タイトル・ヘッダー表示
- t()のキーに対応するja.jsonの値
- コメント・変数名は変更不要（内部識別子はそのままでよい）

---

## 触らないこと

- 草→体調の閾値（3/9の境界はそのまま）
- 💩の増加ロジック（3日に1個、上限4）
- 品質ポイントの加算ロジック（薬草=5pt）

---

## 完成の目安

1. 品質の閾値が30/50/80になっている
2. 💩4個時のペナルティが-5pt/日になっている
3. 画面上に「アップキープ」という文字が表示されなくなっている

---

## 確認してほしいこと（実装後に報告）

- 「アップキープ」の表記が残っている箇所があれば報告すること
- qualityPointのクランプ処理が既存にあったか、新たに追加したか
- 不明点は実装を止めてやましさんに確認すること
