## プロジェクト概要

**ゲームタイトル**：うしリクレイム
**リポジトリ**：yamasinnya/ushireclaim
**公開URL**：https://yamasinnya.github.io/ushireclaim/

大分県で黒毛和牛の繁殖農家を営む「やまうし」が制作している牧場育成×探索ゲーム。
実際の繁殖農家の仕事をベースにしつつ、魔石・魔力などのファンタジー要素を加えた作品。

詳細な設計・仕様は以下のドキュメントを参照：
→ やまやま牧場_設計まとめ.md（同リポジトリ内）

## 技術スタック

- **言語**：HTML + Vanilla JavaScript
- **探索パート**：Phaser.js（後述のバグ注意）またはピュアJS
- **データ保存**：localStorage
- **公開**：GitHub Pages（静的ファイルのみ、サーバー・DBなし）
- **バックアップ**：ふっかつのじゅもん（Base64エンコード、未実装）
- **スタイル**：CSS（フレームワークなし）

やましさんのメインスタックはPHP/Slim4/MySQL/jQueryだが、
このゲームはフロントエンドのみで完結させる方針。

  wrapWara: 0,          // ラップ藁（草ポイント単位で保持。5pt=1日分）
  manaUsed: 0,          // 当日消費した魔力（日付変更でリセット）
  grassStock: 0,        // 当日収集した草ポイント（feeding.htmlでリセット）
  buildings: {
    gyusha_small: true, // 牛舎（小）
    // 将来：gyusha_large, silo, compost, wrap_storage 等
  },
  cows: [
    {
      id: 'cow_001',
      name: 'ふうか',        // メスはひらがな・数字のみ、オスは漢字・数字のみ、最大8文字
      gender: 'female',
      age: 120,             // ゲーム内日数（1年=24日）★フィールド名はage（ageDaysではない）
      seed: 1234,           // ブチ模様の乱数シード
      condition: 7,         // 体調 1〜10
      quality: 2,           // 品質 1=劣 2=可 3=良 4=優
      qualityPoint: 0,      // 品質ポイント（牛ごと。閾値：劣→可30、可→良50、良→優80）
      skill: 'roku',        // スキル識別子
      type: 'mother',       // mother / calf
      pregnantDay: 0,       // 妊娠経過日数（0=非妊娠、18日で出産予定）
      poopCount: 0,         // 💩の数（0〜4、3日に1個増加）
      diseaseAlert: false,  // 😷アイコン表示フラグ（急性鼓腸症）
    }

## 実装済みの主要機能

### 画面フロー
```
title.html → prologue.html → home.html
                                ↓
                    explore.html（神経衰弱）
                                ↓
                    feeding.html → date_change.html → mana_gain.html → upkeep.html
                                ↓
                    barn.html / street.html / map.html
```

### 探索パート（神経衰弱）
- 80枚・3種類（コモン1pt/アンコモン3pt/レア→品質ポイント+5）
- 同じ種類でペア成立
- 魔力（体調から動的計算）の管理
- 翌日は盤面の半分をシャッフル

### スキル一覧
| 識別子 | 表示名 | 効果 |
|---|---|---|
| herdboys_eye | 👦 牧童の目 | ミスマッチ時の表示時間延長（1.5秒） |
| trace | 🐾 刈り跡の記憶 | 前日マッチしたアンコモン以上が翌日自動収穫 |
| roku | 🔮 第六感 | 毎朝アンコモン以上3か所がうっすら光る |
| zenno | （デバッグ用） | 全カード見える。本番では使用しない |

※ 🌀 回転斬りはスキルではなく全牛共通のベース行動。品質から使用回数を算出（rotationSlashMaxUses(quality)）。

### 体調・品質システム
- 草ポイント3以下→体調-1、4〜8→維持、9以上→体調+1
- 体調から魔力を動的計算（conditionToMagic関数）
- 品質ポイント閾値：劣→可30pt、可→良50pt、良→優80pt
- 💩4個で品質ポイント-5pt/日

### 牛舎
- 背景画像の上にスプライトを重ねる構成
- 牛タップ→主人公が歩いて近づく→詳細シート
- 💩タップで床替え（自分でやる：魔力4消費、外注：1000G、1年目半額）
- 😷アイコン（急性鼓腸症アラート、フラグのみ実装、ロジックは未実装）

### ショップ（街）
- 5軒：何でも屋・獣医・コントラクター・建設屋・競り市場
- コントラクター：ラップ藁購入実装済み（6日1200G/12日2400G/24日4800G）
- 競り市場：母牛購入実装済み（固定1頭・100G・牧童の目・3歳・体調6・品質可）
- 他の店舗は準備中

### プロローグ
- 既存スプライト流用（新画像なし）
- シーン配列でセリフ・スプライト状態を管理
- 牛の命名（メス：ひらがな・数字、最大8文字）
- 暗転→タイトルロゴ演出

### 共通ヘッダー
- 全画面共通（explore.htmlを除く）
- 表示項目：年月・Day・魔力・ゴールド・ラップ藁残量

## 実装上の注意（バグ・地雷記録）

### Phaser.js カメラ追従
- `camera.scrollX/Y` の手動計算はズレが生じてキャラが画面外に描画される
- **必ず `camera.startFollow(container)` を使うこと**

### Phaser.js ダイアログ・UI表示
- `setScrollFactor(0)` はズーム倍率の影響を受けるため画面外に描画される
- **ダイアログ・ステータス表示はHTMLのdivオーバーレイで実装すること**

### Phaser.js Graphicsオブジェクト
- `g.x = value` で位置を動かすとカリングが誤作動して非表示になる
- キャラクター描画は `this.add.rectangle()` などのShapeオブジェクトを使うこと

### スキルのマージ処理
- `mergeCowWithDefault` でスキルをcommon.js側で上書きしない（廃止済み）
- セーブ側のスキルを正として扱う

### ローカル確認
- ja.jsonをfetchするため `file://` 直開きでは動作しない
- `npx serve .` 等で簡易サーバーを立ててから確認すること

## 命名ルール（牛の名前）

実際の黒毛和牛の命名規則に準拠：

| 性別 | 使用可能文字 | 最大文字数 |
|---|---|---|
| メス | ひらがな・数字のみ | 8文字 |
| オス | 漢字・数字のみ | 8文字 |

バリデーションは `assets/js/naming.js` の `validateCowName(name, gender)` を使うこと。
将来の子牛命名でも同じ関数を流用する。

## itch.io公開を見据えた実装ルール

将来的にitch.ioでの公開を視野に入れ、以下は**今のうちに**徹底すること。

### 文字列は直書きしない
```js
t("feed_success")  // ja.jsonから取得
```
- 中身は `ja.json` のみで良い（英語版は後回し）

### セーブデータには必ずversionを持たせる
```js
{ version: 1, ... }
```

### 内部名称（変数・プロパティ）は英語で統一
- 画面表示のみ日本語、コード内部は英語

### スマホ縦画面・タップ操作が前提
- 縦画面固定、タップ操作のみ、マウスオーバー不要

### イベント・スキルのJSON化はまだしない
- if文でベタ書きでよい
- 3〜4個同じパターンが溜まってからJSON化を検討

### 優先度が低いもの（現時点では対応不要）
- BGM実装
- 英語辞書（en.json）の中身
- MOD対応
